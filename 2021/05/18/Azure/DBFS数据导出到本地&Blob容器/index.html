  • 直接在工作区中使用脚本，使DBFS导出csv文件到Blob。暂时测试失败，将csv导入到blob中发生错误：Uploads to to public accounts using anonymous access is prohibited.暂时没有找到解决办法。但应该可以解决，只是没有找到更多的相关资料。

1.数据文件转成csv存储再DBFS中

2.下载 Databrick CLI 客户端

通过此客户端，可使用类似于 Unix 命令行中所用的命令与 DBFS 进行交互。

Databrick CLI 官方文档https://docs.microsoft.com/zh-cn/azure/databricks/dev-tools/cli/

安装CLI

设置身份验证，本次测试使用的是 [Databricks 个人访问令牌]设置的身份验证

  • 单击“用户配置文件”图标 它位于 Azure Databricks 工作区的右上角。
  • 单击“用户设置”。
  • 转到“访问令牌”选项卡。
  • 单击“生成新令牌”按钮。
  • 选择性地输入说明（注释）和有效期。

  • 注意，生成以后及时拷贝保存，key只有在生成的时候能够看到

windows配置CLI

  • cmd运行 databricks configure --aad-token。 此命令发出提示：
  • 按照 https://adb-<workspace-id>.<random-number>.azuredatabricks.net 的格式输入每工作区 URL

  • 该命令继续发出提示，要求你输入个人访问令牌：

完成提示后，访问凭据存储在 ~/.databrickscfg（Unix、Linux、macOS 上）或 %USERPROFILE%\.databrickscfg 文件（Windows 上）中。 该文件包含默认配置文件条目：
